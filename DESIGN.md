# DESIGN: B2B/B2G 플랫폼 디자인 요구사항

작성일: 2026-07-10
목적: UI/UX 디자인 제작을 위한 요구사항 정의
관련 문서: PRD.md (제품 요구사항)

## 1. 제품 맥락 (디자이너가 알아야 할 것)

### 1.1 무엇을 만드는가
한국 제조사/공급사와 해외 바이어를 잇는 B2B/B2G 커뮤니티 플랫폼. 제품 카탈로그(판매글)와 수요 공고(요청글)가 중심이며, 모든 회원 간 접촉은 관리자가 중개한다.

취급 영역은 두 축이다.
- 주력: 산업용(Industrial), EPC+IPP (플랜트/발전)
- 병행: Commercial (화장품, 건강기능식품, 의류, 기타 잡화 등 소비재)

중요: 소비재를 함께 다루므로 전체 디자인이 무거운 산업용 무드(짙은 네이비, 그레이 일변도, 금속 질감)로 기울면 안 된다. 밝고 깨끗한 중립 베이스 위에서, 게시판별 무드 차이는 제품 이미지 콘텐츠가 만들도록 한다.

### 1.2 사용자
- 해외 바이어: 영어 사용, 시차 존재, 모바일 접속 비중 높음. 신뢰할 수 있는 공급처를 찾는 것이 목적
- 한국 제조사/공급사: 중소기업 다수, 자체 홈페이지가 없거나 관리 못하는 경우 많음. 해외 판로 확보가 목적
- 관리자: 모든 검수/승인/설정의 주체. 운영 효율이 곧 플랫폼 품질
- 코디네이터: 관리자가 지정한 특수 역할. 자기가 추천한 회원을 관리/지원

### 1.3 디자인이 풀어야 할 핵심 과제
1. 신뢰의 시각화: 배지(제조사, 인증)가 목록/상세/프로필 어디서든 즉시 인지되어야 한다. 산업용 거래는 신뢰가 전부다
2. 대기의 투명화: 문의와 게시글은 관리자 검수를 거치므로 대기가 발생한다. 진행 단계를 택배 조회처럼 보여주어 대기를 불안이 아닌 과정으로 만든다
3. 잠금의 유혹화: 비회원에게 상세는 그라데이션으로 잘려 보인다. "여기 더 있다"는 것을 증명하며 가입을 유도하는 연출이 필요하다
4. 멈춤 없는 체감: 모든 이동/행동/검색에 스켈레톤과 로딩 표시. 멈춰 있다는 느낌 제거

### 1.4 언어와 텍스트
- UI 기본 언어: 영어. 한국어 지원 (i18n)
- 모든 UI 텍스트는 언어팩 키로 관리되므로, 디자인 시 텍스트 길이 변화(영/한)를 견디는 레이아웃 필요
- 이모지 사용 금지 (UI 전체)

## 2. 디자인 원칙

### 2.0 디자인 컨셉: 토스 스타일 참조
참조 대상은 토스(앱인토스) 디자인 가이드의 원칙과 톤이다. 토스의 그래픽 자산과 TDS 컴포넌트 자체는 비바리퍼블리카의 지식재산이므로 사용하지 않는다. 참조하는 것은 다음 원칙들이다.

- 단순하고 명료하며 깨끗한 디지털 그래픽 스타일. 손그림, 서정적 화풍, 만화적 표현 배제
- 밝고 정돈된 인상. 모노톤 과다, 뿌옇고 칙칙한 인상 배제. 중간 명도의 색감
- 한 화면에 핵심 그래픽은 하나. 비슷한 크기의 그래픽 병렬 배치로 시선 분산 금지
- 장식적 효과 배제: 의미 없는 묘사, 파티클, 과한 그라데이션 금지 (비회원 잠금의 페이드아웃은 기능적 연출이므로 예외)
- 그래픽은 장식이 아니라 화면의 의미 이해를 돕는 역할. 문맥에 맞게, 정보 밀도에 맞는 크기로
- CTA는 버튼 라벨만 보고 다음 행동을 예측할 수 있어야 한다. 모호한 라벨, 가치 설명 반복 금지
- 다크패턴 금지: 진입 즉시 전면을 가로막는 인터럽트 금지 (본 플랫폼의 하단 배너 원칙과 일치), 나갈 수 없는 선택지 구조 금지, 부정적이거나 호소하는 감정 표현 금지
- 상황을 정확히 전달: 오류가 아닌데 경고 아이콘 금지, 대기가 없는데 로딩 표시 금지
- 기준 해상도 중심 설계: 하나의 논리 해상도(세로형 약 360x640 ~ 420x740 범위)를 기준으로 스케일링 대응. 기기별 다른 레이아웃 운영 금지
- UX 라이팅 톤 (영어 기준으로 번안 적용): 능동형 문장, 긍정형 문장 우선, 캐주얼하되 명확한 어조. 한국어 UI는 해요체로 통일

### 2.1 모바일 퍼스트
- 기준 캔버스: 작은 모바일뷰 (약 360-390px)
- 반응형 범위: 작은 태블릿 ~ 작은 모니터 PC까지 깨지지 않는 프레임
- 체감 목표: 웹이 아니라 앱처럼 (PWA로 설치되는 것을 전제)

### 2.2 앱 표준 인터랙션
- 당겨서 새로고침 (pull to refresh)
- 갤러리형 레이아웃의 좌우 스와이프
- 스켈레톤 UI: 모든 로딩 구간
- 로딩 인디케이터: 버튼 액션, 페이지 전환, 검색

### 2.3 행동 안전장치
- 모든 행동 버튼은 2중 안내 후 확인 (확인 모달 패턴 통일 필요)
- 삭제는 2단계 (휴지통 경유 후 완전삭제)
- 작성 중 이탈 시 경고

### 2.4 배지 시각 체계
- 제조사 배지(역할 신호)와 인증 배지(신원 신호)의 시각 구분
- 중복 표시 시 배치 규칙 (프로필, 목록 카드, 상세, 메시지)
- 배지 종류는 관리자가 추가 가능하므로 확장 가능한 배지 컴포넌트로 설계

### 2.5 상태 표시 체계
- 게시글: 임시저장 / 검토 중 / 승인(게시됨) / 반려 / 마감(요청글)
- 문의: 발송됨 / 관리자 검토 중 / 전달됨 / 답변 작성됨 / 답변 검토 중 / 답변 도착
- 구독: 활성 / 만료 예정 / 만료
- 상태별 색상과 아이콘 체계를 전역으로 통일

## 3. 전역 공통 컴포넌트

| 컴포넌트 | 요구사항 |
|---|---|
| 헤더 | 로고, 동적 메뉴(관리자가 추가/순서변경/숨김 가능하므로 개수 가변 대응), 검색, 알림 종 아이콘(미확인 수 뱃지), 프로필/로그인 |
| 푸터 | 법적 페이지 링크, 언어 전환 |
| 알림 종 | 미확인 수 표시, 클릭 시 알림센터 |
| 쿠키 배너 | 첫 방문 시, 필수/선택 구분 동의. 설치 배너보다 우선 노출 |
| PWA 설치 배너 | 하단 배너, 방해 최소화, 닫으면 일정 기간 숨김. iOS용 안내 변형(공유 → 홈 화면에 추가) |
| 2중 확인 모달 | 모든 행동 버튼 공통 패턴 |
| 스켈레톤 세트 | 목록 카드, 상세, 대시보드, 메시지 스레드 각각 |
| 토스트/알림 | 액션 결과 피드백 |
| 그라데이션 잠금 | 비회원 상세 하단 페이드아웃 + 로그인 유도 CTA |
| 빈 상태(Empty state) | 글 없는 목록, 알림 없음, 검색 결과 없음 등 |
| 배지 컴포넌트 | 2.4 참조 |
| 상태 라벨 | 2.5 참조 |
| 언어 전환 | 영어/한국어 |

### D. 관리자 영역 (통합 관제)

원칙: 관리자 대시보드는 프로젝트의 모든 기능을 관리하는 통합 관제 화면이다. 사이드 네비게이션 등으로 아래 전 영역에 접근 가능해야 하며, 모든 정책 변경이 코드 수정 없이 이 화면에서 완결된다. 관리자 영역은 데이터 밀도가 높으므로 모바일 퍼스트 원칙의 예외로, 데스크톱 우선 + 모바일 대응으로 설계한다.

| # | 페이지 | 핵심 요소 |

## 5. 디자인 우선순위 (제작 순서 제안)

1. 전역 컴포넌트와 디자인 시스템 (3장): 색상/타이포/배지/상태 라벨/스켈레톤/모달. 이것이 전 페이지의 어휘가 된다
2. 공개 영역 핵심 흐름: 랜딩 → 게시판 목록 → 상세(비회원 그라데이션) → 가입 유도. 첫 방문 바이어의 여정
3. 회원 핵심 흐름: 가입 → 글 작성(에디터) → 검수 상태 확인 → 문의 수신/응답. 공급사의 여정
4. 문의 중개 흐름: 문의 발송 → 6단계 진행 표시 → 답변 도착. 플랫폼 가치의 심장
5. 대시보드/배지/구독/미니홈피
6. 관리자 영역
7. 코디네이터 영역

## 6. 제약과 주의사항

- 연락처/이메일이 노출되는 화면은 존재하지 않는다 (관리자 화면과 코디네이터의 직접 추천 회원 목록 제외)
- 문의는 실시간 채팅이 아니다: 모든 문의/답변은 관리자 검수를 거치는 비동기 서신 교환이다. 말풍선 + 하단 고정 입력창 + 즉시 전송의 채팅 UI를 사용하지 않는다. 이메일/티켓 시스템의 은유를 따른다 (문의 건 카드, 진행 상태 타임라인, 답변 작성 버튼). 유일한 예외는 코디네이터 직접 메시지 채널(C9)로, 이것만 검수 없는 직접 전달이지만 이 역시 메시지 단위 전송이며 실시간 접속을 전제하지 않는다
- 상세 페이지의 비회원 잠금은 시각 연출일 뿐, 잠긴 데이터는 실제로 전달되지 않는 전제로 설계 (스크롤로 우회 불가한 구조)
- 메뉴는 동적(개수/순서 가변)이므로 고정 폭 네비게이션 금지, 넘칠 때 처리 규칙 필요
- 동영상 자동 재생은 음소거 상태가 상한선. 목록에서는 썸네일만, 상세에서만 플레이어
- 영어 기준으로 디자인하되 한국어 전환 시 길이 변화 검증
- 카카오톡 인앱 브라우저에서 열람 화면이 깨지지 않아야 함 (dvh, fixed 최소화)
- 이모지 사용 금지, 아이콘은 아이콘 폰트/SVG 세트로 통일

## 7. 미확정 (디자인 진행 중 확정 필요)

1. 랜딩 페이지 세부 구성 (섹션 순서, 추천 노출 영역 위치)

4. 추천 노출 로테이션 규칙 (추천 영역 카드 수와 순환)
