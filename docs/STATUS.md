## UX 컨벤션 (모든 화면 공통, 필수)

1. 모든 항목은 **보기 → 수정하기/추가하기 → (취소|저장)** 흐름. 목록 클릭 = 상세/보기,
   편집은 명시적 버튼 뒤에만 존재
2. 편집/작성 화면에는 반드시 취소 + 저장 버튼. 변경사항이 있는 상태의 취소는 2중 확인
   (`EditFormFrame` / `DiscardModal` 사용)
3. 대시보드에 역할별 행동 버튼 노출 (제품 등록하기, 요청 올리기, 내 글 관리 등)
4. 파괴적/불가역 행동은 `ConfirmSubmit` 2중 확인
5. 푸시 전 게이트: eslint 0 error → next build 통과 → 라우트 QA 스모크 (익명 + 로그인 세션)
관련 문서: PRD.md, DESIGN.md

## 인증 UX 확정 사항

- 재설정 링크 = 1회용. 링크 세션은 비밀번호 저장 전까지 다른 페이지 차단, "취소하고 기존 비밀번호 유지" 탈출구 제공
- 사용된 링크 재클릭: 상태별 분기 (잠금 중→변경 화면+안내 / 완료 후→대시보드 / 비로그인→로그인+만료 안내)
- `/reset/update`는 재설정 링크 경유로만 접근 가능 (서버 액션도 동일 검사)
- 입력 필드: X 지우기 + 비밀번호 눈 토글 (`components/ui/TextField.tsx` — 신규 폼 필수 사용)
- 로그인 상태 유지: 체크박스 (기본 해제 = 브라우저 종료 시 로그아웃)
